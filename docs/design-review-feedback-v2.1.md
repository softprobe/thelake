# Design Review Feedback v2.1 - Iceberg Raw Table + ClickHouse Metrics

**Date**: 2025-01-31  
**Document**: `docs/migration-to-iceberg-design.md`  
**Version**: 2.0 → 2.1

---

## Summary

Captured the latest decisions: raw OTLP data is appended directly to an Iceberg table (with multi-session Parquet files and row offsets), while Flink populates ClickHouse time-series tables for fast statistics queries. Metadata schema updated to store `(file_path, row_offset, row_count)` per session segment.

---

## Key Changes

1. **Iceberg raw table** – documented schema, partitioning, and session grouping within Parquet files.
2. **Session segment indexing** – metadata table now records row offsets/counts for efficient replay.
3. **ClickHouse metrics tables** – operation counts and tag counts generated by Flink and stored in SummingMergeTree tables for low-latency analytics.
4. **Flush strategy** – clarified writer batching (multiple sessions per file, row-group alignment).
